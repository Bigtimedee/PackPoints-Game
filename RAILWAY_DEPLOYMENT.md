# PackPTS Railway Deployment Guide

## 🚀 Step-by-Step Deployment

### 1. Prepare Railway Project

1. **Sign up at [Railway.app](https://railway.app)**
2. **Create a new project** → "Deploy from GitHub repo"
3. **Connect your GitHub account** and select the PackPoints-Game repository

### 2. Add PostgreSQL Database

1. In your Railway project, click **"+ New"** → **"Database"** → **"PostgreSQL"**
2. Railway will automatically:
   - Create a Postgres instance
   - Generate a `DATABASE_URL` environment variable
   - Make it available to your app

### 3. Configure Environment Variables

#### Option A: Use Railway Dashboard (Recommended)
1. Click on your service → **"Variables"** tab
2. Click **"RAW Editor"**
3. Copy ALL variables from `.env.railway.example`
4. Replace placeholder values with your actual credentials

#### Option B: Use Railway CLI
```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link to your project
railway link

# Add variables one by one
railway variables set NODE_ENV=production
railway variables set APP_URL=https://your-app.railway.app
# ... etc
```

### 4. Critical Environment Variables

**Must configure before first deployment:**

```bash
# Database (auto-generated by Railway)
DATABASE_URL=${{Postgres.DATABASE_URL}}

# Stripe (REQUIRED - use direct keys)
STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_WEBHOOK_SECRET_LIVE=whsec_...

# WorkOS
WORKOS_API_KEY=sk_...
WORKOS_CLIENT_ID=client_...

# Security salts (generate random strings)
IP_HASH_SALT=<random-32-char-string>
DEVICE_HASH_SALT=<random-32-char-string>
FOUNDERS_PASS_PEPPER=<random-32-char-string>

# App URL (get from Railway after first deploy)
APP_URL=https://your-app-name.up.railway.app
APP_BASE_URL=https://your-app-name.up.railway.app
BASE_URL=https://your-app-name.up.railway.app
```

### 5. Deploy

1. **Commit the new config files:**
```bash
cd ~/Projects/PackPoints-Game
git add railway.json nixpacks.toml .env.railway.example RAILWAY_DEPLOYMENT.md
git commit -m "Add Railway deployment configuration"
git push
```

2. **Railway will automatically:**
   - Detect the push
   - Install dependencies (`npm ci`)
   - Run build (`npm run build`)
   - Start the server (`npm run start`)

3. **Watch the build logs** in Railway dashboard

### 6. Run Database Migrations

After first successful deployment:

```bash
# Using Railway CLI
railway run npm run db:push

# OR via Railway dashboard
# Go to your service → Settings → Deploy Trigger → Add "npm run db:push && npm run start"
```

### 7. Configure Stripe Webhooks

1. **Get your Railway URL**: `https://your-app-name.up.railway.app`

2. **In Stripe Dashboard:**
   - Go to **Developers** → **Webhooks**
   - Click **"Add endpoint"**
   - URL: `https://your-app-name.up.railway.app/webhooks/stripe`
   - Select events:
     - `checkout.session.completed`
     - `customer.subscription.created`
     - `customer.subscription.updated`
     - `customer.subscription.deleted`
     - `invoice.payment_succeeded`
     - `invoice.payment_failed`
   - Copy the **Signing secret** → Add to Railway as `STRIPE_WEBHOOK_SECRET_LIVE`

3. **Update environment variables** with the webhook secret

### 8. Set Up Custom Domain (Optional)

1. In Railway dashboard → Your service → **Settings** → **Domains**
2. Add custom domain: `packpts.com`
3. Follow DNS configuration instructions
4. Update environment variables:
```bash
APP_URL=https://packpts.com
APP_BASE_URL=https://packpts.com
BASE_URL=https://packpts.com
```

## ⚠️ Common Issues & Fixes

### Issue 1: Build Fails - "Cannot find module 'tsx'"

**Fix:** Ensure `tsx` is in `dependencies`, not `devDependencies`

Already fixed in `package.json` ✅

### Issue 2: "Stripe connection not found"

**Cause:** Replit-specific Stripe connector doesn't work on Railway

**Fix:** Use direct Stripe API keys instead of connector:
```bash
STRIPE_SECRET_KEY=sk_live_...  # NOT using connector
STRIPE_PUBLISHABLE_KEY=pk_live_...
```

The app will automatically fall back to environment variables.

### Issue 3: Database Connection Error

**Symptoms:**
```
Error: Connection terminated
DATABASE_URL is not set
```

**Fix:**
1. Ensure PostgreSQL service is added to Railway project
2. Check that `DATABASE_URL` variable exists (Railway auto-generates this)
3. Format should be: `postgresql://user:password@host:port/database`

### Issue 4: Port Binding Error

**Symptoms:**
```
Error: listen EADDRINUSE: address already in use :::5000
```

**Fix:** Railway provides a random `PORT` variable. The app already handles this:
```typescript
const port = parseInt(process.env.PORT || "5000", 10);
```

No changes needed ✅

### Issue 5: Static Files Not Served

**Symptoms:** Images/assets return 404

**Fix:** Ensure `dist/public` exists after build. Check `script/build.ts`:
```typescript
// Should copy public folder to dist/public
```

Already handled ✅

### Issue 6: FFmpeg Not Found

**Symptoms:**
```
Error: Cannot find ffmpeg binary
```

**Fix:** The `nixpacks.toml` includes `ffmpeg-full`:
```toml
nixPkgs = ["nodejs-20_x", "ffmpeg-full"]
```

This installs FFmpeg system-wide. The app uses `ffmpeg-static` as fallback.

### Issue 7: Session Store Errors

**Symptoms:**
```
Error: connect-pg-simple requires a session store
```

**Fix:** Ensure `DATABASE_URL` is set and sessions table is created:
```bash
railway run npm run db:push
```

### Issue 8: Missing Environment Variables

**Symptoms:** Features not working (growth agent, email, etc.)

**Fix:** Copy ALL variables from `.env.railway.example` to Railway dashboard.

**Check which are missing:**
```bash
# Add this to server/index.ts temporarily
const requiredEnvVars = [
  'DATABASE_URL',
  'STRIPE_SECRET_KEY',
  'WORKOS_API_KEY',
  'GMAIL_USER',
  'OPENAI_API_KEY'
];

requiredEnvVars.forEach(varName => {
  if (!process.env[varName]) {
    console.error(`❌ Missing required env var: ${varName}`);
  }
});
```

## 🔧 Railway-Specific Adjustments Needed

### 1. Remove Replit-Specific Code

The Stripe connector code needs to be updated to skip Replit connector on Railway:

**File: `server/stripeClient.ts`**

Find this section (around line 20):
```typescript
const isReplitDeployment = process.env.REPLIT_DEPLOYMENT === '1';
```

Add Railway detection:
```typescript
const isRailway = process.env.RAILWAY_ENVIRONMENT !== undefined;
const isReplitDeployment = process.env.REPLIT_DEPLOYMENT === '1';

// Skip Replit connector on Railway
if (isRailway) {
  console.log('[Stripe] Railway environment detected, using direct env vars');
  // Skip to direct environment variable section
}
```

### 2. Update Session Store Configuration

**File: `server/index.ts`** (if not already Railway-compatible)

Ensure session store uses `DATABASE_URL` directly:
```typescript
import pgSession from 'connect-pg-simple';
import session from 'express-session';

const pgStore = pgSession(session);

app.use(session({
  store: new pgStore({
    conString: process.env.DATABASE_URL,
    tableName: 'session'
  }),
  secret: process.env.SESSION_SECRET || 'change-this-secret',
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: process.env.NODE_ENV === 'production',
    maxAge: 30 * 24 * 60 * 60 * 1000 // 30 days
  }
}));
```

## 🆘 Still Having Issues?

### Enable Debug Logging

Add to Railway environment variables:
```bash
DEBUG=*
LOG_LEVEL=debug
```

### Check Railway Logs

```bash
railway logs
```

Or in dashboard: Your service → **Deployments** → Click deployment → **View Logs**

### Test Database Connection

```bash
railway run node -e "const {Pool}=require('pg'); const p=new Pool({connectionString:process.env.DATABASE_URL}); p.query('SELECT NOW()').then(r=>console.log('✅ DB connected:',r.rows[0])).catch(e=>console.error('❌ DB error:',e))"
```

### Verify Environment Variables

```bash
railway run node -e "console.log('DATABASE_URL:', process.env.DATABASE_URL ? '✅ Set' : '❌ Missing'); console.log('STRIPE_SECRET_KEY:', process.env.STRIPE_SECRET_KEY ? '✅ Set' : '❌ Missing');"
```
