Implement the “PackPTS Redemption Financial Guardrail Engine” to ensure PackPts Inc. NEVER loses money on redemptions applied to external marketplace purchases (eBay, Goldin). This must enforce margin-backed redemptions using a company-side margin pool + transaction contribution logic.

STACK (do not change)
Frontend: React 18 + Vite + TypeScript + Tailwind + shadcn/ui + React Query + Wouter
Backend: Express + TypeScript + Passport + express-session
Database: Postgres (Neon) + Drizzle ORM
Payments: Stripe
Marketplace Integrations: eBay + Goldin (existing)

CORE FINANCIAL RULE (HARD INVARIANT)
For any redemption:

credit_value_usd <= available_margin_pool + transaction_margin

If this condition fails:
- Clamp redemption down to allowed maximum OR
- Reject redemption entirely

At no time may PackPTS create a negative contribution margin event.

---

SECTION A — COMPANY MARGIN LEDGER (TREASURY)

Create new tables:

1) margin_ledger (company-side accounting, NOT user-visible)
- id uuid pk
- source_type enum('PACKPTS_SALE','AFFILIATE_PAYOUT','PARTNER_REBATE','MANUAL_ADJUSTMENT')
- amount_cents int (positive only)
- reference_id text nullable
- created_at timestamp

2) margin_usage
- id uuid pk
- redemption_id uuid
- amount_cents int
- created_at timestamp

3) redemption_reservations (prevent race conditions)
- id uuid pk
- purchase_intent_id uuid
- reserved_cents int
- status enum('ACTIVE','RELEASED','CONSUMED')
- created_at timestamp

---

SECTION B — STEP 1: Compute Redemption Cost

When user applies PackPTS:

credit_value_cents =
  packpts_requested * PACKPTS_USD_VALUE_CENTS

This is the economic “cost” of this redemption.

Store this value in external_purchase_intent.

---

SECTION C — STEP 2: Compute Available Margin Pool

Implement function getAvailableMargin():

total_margin =
  SUM(margin_ledger.amount_cents)

total_used =
  SUM(margin_usage.amount_cents)

total_reserved =
  SUM(redemption_reservations.reserved_cents WHERE status='ACTIVE')

available_margin_pool =
  total_margin - total_used - total_reserved

If available_margin_pool < 0, treat as 0 and log alert.

---

SECTION D — STEP 3: Compute Transaction Margin

For each purchase intent:

transaction_margin_cents =
  affiliate_rate * haircut * purchase_price_cents

Rules:
- Default haircut = 0.5 (50%)
- Goldin default affiliate margin = 0 unless explicitly configured
- Store per-source rates in marketplace_margin_config table:

marketplace_margin_config:
- source enum('ebay','goldin')
- affiliate_rate decimal
- haircut decimal
- updated_at

---

SECTION E — STEP 4: Redemption Approval Gate

When POST /api/marketplace/redemption/apply is called:

Run inside DB transaction:

1) Recompute:
   - credit_value_cents
   - available_margin_pool
   - transaction_margin_cents

2) Compute allowed_redemption_cap:

allowed_cap =
  available_margin_pool + transaction_margin_cents

3) Clamp:

approved_credit =
  MIN(credit_value_cents, allowed_cap)

If approved_credit <= 0:
  - Reject redemption
  - Return "INSUFFICIENT_MARGIN_BACKING"

4) Convert approved_credit back to PackPTS:

approved_packpts =
  FLOOR(approved_credit / PACKPTS_USD_VALUE_CENTS)

5) Reserve margin immediately:

Insert redemption_reservations row:
- purchase_intent_id
- reserved_cents = approved_credit
- status = ACTIVE

6) Spend user PackPTS:

Insert ledger SPEND entry with idempotency key:
"redeem:" + purchase_intent_id

7) Save approved values on purchase intent:
- approved_packpts
- approved_credit_cents
- status = APPROVED

---

SECTION F — Redemption Finalization

When purchase is confirmed (affiliate callback OR manual admin confirmation):

1) Mark redemption_reservations as CONSUMED
2) Insert margin_usage record:
   - redemption_id
   - amount_cents = approved_credit_cents
3) Mark purchase intent as CREDIT_GRANTED

If purchase is canceled/expired:

1) Release reservation:
   - redemption_reservations.status = RELEASED
2) Roll back PackPTS SPEND by issuing ledger ADJUST credit
