This must be production-grade and impossible to “drift” over time. The solution must be ledger-based (append-only), idempotent, and server-authoritative. Never rely on a client-calculated balance. Never mutate a single “points” field without an immutable audit trail.

──────────────────────────────────────────────────────────────────────────────
CORE PRINCIPLE (NON-NEGOTIABLE)
PackPTS balance = SUM(ledger credits) - SUM(ledger debits)
- Ledger is append-only. No updates/deletes except via admin reversal entries.
- Every credit/debit must have a unique idempotency key so it can be safely retried.
- UI reads balance from ONE authoritative API endpoint.
- Store purchases credit via Stripe webhooks only.
- Gameplay credits only when a match round is finalized (server-side) and only once.

B) SERVER: SINGLE AUTHORITATIVE “CREDIT/DEBIT” SERVICE
Create a server module: `lib/packpts/ledgerService.ts` with these functions:

1) `applyLedgerEntry(params)`:
Inputs:
- userId
- direction
- amountPackpts
- source
- eventType
- refType
- refId
- idempotencyKey
- metadata

Behavior:
- Validate user exists, amount positive integer.
- Start DB transaction.
- Insert into packpts_ledger using idempotencyKey as unique guard.
  - If UNIQUE violation on idempotencyKey: treat as success (return existing entry) (idempotent).
- Update packpts_balances atomically:
  - If row exists: balance += (direction == credit ? +amount : -amount), version += 1
  - If no row: create it with starting balance (credit=amount, debit=0-amount but DO NOT allow negative if your policy forbids).
- Commit.
- Return new balance and ledger entry.

2) `getBalance(userId)`:
- Prefer packpts_balances row if exists
- Else compute SUM(ledger) and optionally seed cache
- Return:
  - balancePackpts
  - lifetimeEarnedPackpts (sum credits)
  - lifetimeSpentPackpts (sum debits)

3) `reconcileBalance(userId)` (admin-only utility):
- Compute from ledger, compare to cache, fix cache if mismatch, log event.

Add unit tests for idempotency and concurrency (two simultaneous credits).
