MISSION-CRITICAL BUG: PLAYER NAME IS VISIBLE ON CARD IMAGE (GAME UNPLAYABLE)
You are fixing PackPTS so the player’s printed name on the trading card image is ALWAYS masked/hidden before the user answers. This must work on mobile Safari (iPhone) and desktop, and must be set-aware (different sets have nameplates in slightly different places).

DO NOT “assume” Card Hedge will provide masked images. PackPTS must mask the image itself in the UI (and optionally via proxy).

========================================================
GOAL
1) Mask the player name area on every card image during gameplay (1v1, 1vRandom, 1vFriends, solo).
2) The mask must be applied BEFORE the image is shown to the user (no flash of unmasked content).
3) The mask must be configurable per card set (because different sets place the nameplate differently).
4) Provide safe defaults that make the game playable immediately (even without per-set tuning).
5) Prevent zoom/reveal tricks (pinch-zoom / long-press / open image in new tab) as best as possible.

========================================================
A) DATA MODEL: ADD MASK REGIONS PER SET
Create/extend a table to store mask configuration per set (or per “set_type”):

card_set_masks
- id uuid pk
- set_key TEXT UNIQUE              // e.g. "1987 Topps Baseball"
- provider_set_id TEXT NULL        // optional mapping if we have it
- mask_version INT DEFAULT 1
- regions JSONB NOT NULL           // array of {xPct, yPct, wPct, hPct, radiusPct, type}
- created_at, updated_at

Where each region is percentages relative to the rendered image box:
- xPct, yPct, wPct, hPct are 0-100
- type: "solid" | "blur" | "pixelate" (start with SOLID)
- radiusPct: optional rounded corners

SEED DEFAULT (global fallback) for unknown sets:
- region 1: { xPct: 0, yPct: 70, wPct: 100, hPct: 30, type: "solid", radiusPct: 0 }
This masks the bottom 30% of the card (where the name almost always appears). It’s aggressive but makes gameplay possible.

Also create a secondary fallback for “slabbed” images where the name may sit slightly higher:
- region 2: { xPct: 0, yPct: 62, wPct: 100, hPct: 38, type: "solid" }

========================================================
B) BACKEND API: FETCH MASK CONFIG FOR A SET
Add endpoint:
GET /api/card-sets/:setKey/mask

Returns:
{
  setKey,
  regions: [...],
  mask_version
}

Logic:
- Lookup by set_key
- If missing: return DEFAULT fallback regions listed above

Cache in-memory for 10 minutes.

========================================================
C) FRONTEND: MASK OVERLAY COMPONENT (NO FLASH)
In the gameplay card view, NEVER render the raw image alone.
Create a component: MaskedCardImage.tsx

Props:
- src (already proxied image URL)
- setKey
- alt
- className

Behavior:
1) Load mask config first (or concurrently), and only then render the image + overlays.
2) Use a wrapper with position:relative.
3) Render the image with:
   - user-select: none
   - -webkit-user-drag: none
   - pointer-events: none on the <img> itself (so long-press/save is harder)
4) Render overlay regions as absolutely positioned divs using percentage coords.
5) Ensure overlays sit ABOVE the image and intercept touches.
6) Make overlays opaque (solid) by default:
   - background: #0b0f16 (match app background)
   - optionally add subtle noise/gradient so it looks intentional
7) IMPORTANT: The overlay must cover even while the image is still loading.
   - Render overlays immediately, then fade-in the image beneath once loaded.

Pseudo:
<Wrapper>
  <Img ... onLoad setLoaded />
  {regions.map(r => <div style={{left:`${r.xPct}%`, top:`${r.yPct}%`, width:`${r.wPct}%`, height:`${r.hPct}%`}} />)}
</Wrapper>
