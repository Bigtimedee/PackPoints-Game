Implement a Provider-Agnostic “Admin Set Importer” so PackPTS Admins can create and populate complete sets for Baseball, Basketball, Football, and Hockey even when CardHedge category filters don’t return results. The importer must:
- Allow Admin to manually define a set (sport/year/brand/set name + keywords)
- Run a diagnostic query against CardHedge to test coverage for that sport/category
- Import cards using set/keyword search (NOT only category)
- Deduplicate cards and store the entire set locally in Postgres
- Track import progress and surface errors to Admin
- Support future providers (eBay, other catalogs) via adapter interface

STACK (do not change)
React 18 + Vite + TS + Tailwind + shadcn/ui + TanStack Query + Wouter
Backend: Express + TS (tsx)
DB: Postgres (Neon) + Drizzle ORM
Auth: Passport + express-session
Card Data Provider: CardHedge

D) IMPORT STRATEGY (SET/KEYWORD FIRST, CATEGORY OPTIONAL)

Create:
src/services/catalog/importer/setImporter.ts

Function:
importSetFromCardhedge(setId: string): Promise<void>

Algorithm:
1) Load set definition from card_sets.
2) Run diagnoseCoverage(sport). Store result to job logs.
3) Build query plan:
   - queryStrings = [
       `${year} ${brand ?? ''} ${setName}`.trim(),
       ...keywords,
       `${setName}` // last resort
     ]
4) For each queryString:
   - Fetch pages sequentially (page 1..N), stopping when:
     - you’ve linked enough cards to be “complete” OR
     - pages exhausted OR
     - you detect diminishing returns (e.g., 3 pages with 0 new inserts)
   - Use request body:
     { search: queryString, page, page_size: 100, category: categoryIfWorksElseNull }
5) For every returned card:
   - Upsert into cards (unique by provider+provider_card_id)
   - Link into card_set_cards
6) Deduplicate:
   - Do not link duplicates; maintain counters
7) Completion heuristic (because “complete set size” may be unknown):
   - Mark SUCCEEDED if:
     - cards_linked >= admin_expected_count (if provided later), OR
     - cards_linked >= 150 and no new cards found in last 3 pages across all query strings (tunable)
   - Else PARTIAL
8) Persist job progress continuously:
   - pages_fetched, cards_found, cards_inserted, cards_linked

Also add (optional) “expected_card_count” column to card_sets so Admin can define completeness explicitly.

- For “Card Set” dropdown, load from local `card_sets` where is_active=true AND has at least X linked cards.
