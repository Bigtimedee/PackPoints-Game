GOAL
Add an Image Search workflow that lets PackPTS submit either an image_url or image_base64 to CardHedge and receive visually similar card matches. This will be used to (a) recover when a card image fails to load, (b) validate that an image is a real card image (not placeholder), and (c) allow “Try Different Card” to fetch a replacement card WITHOUT advancing match progress.

CARDHEDGE ENDPOINT
POST https://api.cardhedger.com/v1/cards/image-search
Headers:
- Content-Type: application/json
- X-API-Key: <secret>

Body (application/json):
- image_base64 (optional) Base64 encoded image (with or without data URI prefix)
- image_url (optional) Public URL to card image
- k (optional) int 1..50 default 10

Expected success response example:
{
  "success": true,
  "results": [
    {
      "similarity": "95.23",
      "distance": 0.0477,

        "card_id": "1587446850514x224832321163624450",
        "description": "Ken Griffey Jr. 1989 Upper Deck Baseball",
        "player": "Ken Griffey Jr.",
        "set": "1989 Upper Deck Baseball",
        "number": "1"
      }
    }
  ],
  "total_results": 5,
  "query_id": "query_abc123",
  "has_cardhedge_matches": true
}

ERRORS TO HANDLE
- 400: missing image input
- 413: image too large (max 10MB)
- 422: validation error
- 502: external api error
- 503: service not configured/unavailable
