Implement “Dual Price Display + PackPTS Apply Prompting” on Marketplace listings so users clearly see:
1) The raw Buy Now price in USD from eBay/Goldin
2) The improved price if they apply PackPTS: “{pts} PackPTS + ${usd}”
This must be computed server-side by the redemption quote engine (treasury-backed), not by the frontend, and must never allow PackPTS to exceed allowed discounts or margin rules.

STACK (do not change)
React 18 + Vite + TypeScript + Tailwind + shadcn/ui + TanStack Query + Wouter
Backend: Express + TypeScript
DB: Postgres (Neon) + Drizzle ORM
Auth: Passport + express-session
Marketplace providers: eBay + Goldin
Wallet: ledger-based (available/pending/locked/debt)
Treasury: margin pool + reservations

========================================================
A) DATA CONTRACTS

Normalized listing already returned by /api/marketplace/listings:
{
  provider: 'ebay'|'goldin',
  externalId: string,
  title: string,
  priceCents: number,
  currency: 'USD',
  imageUrl?: string,
  itemUrl: string,
  sport?: string,
  year?: number,
  setName?: string,
  player?: string,
  raw: any
}

We need a quote structure:
{
  listing: { provider, externalId },
  cashPriceCents: number,
  ptsRequested: number,              // what user wants to apply (optional)
  ptsMaxApplicable: number,          // max allowed by rules
  ptsApplied: number,                // server-chosen (min(requested, max))
  usdDueCents: number,               // remaining cash due if ptsApplied used
  usdSavingsCents: number,           // how much USD is reduced vs cash price
  effectiveValuePerPtMicrousds: number,
  reasons: string[],                 // explain clamps/limits (optional)
  ctaLabel: string                   // e.g., "Apply PackPTS"
}

========================================================
B) BACKEND — QUOTE ENGINE ENDPOINTS (SERVER-SIDE ONLY)

1) GET /api/wallet/balance
Return:
{
  availablePts: number,
  pendingPts: number,
  lockedPts: number,
  debtPts: number,
  status: 'NORMAL'|'RESTRICTED'|'FROZEN'
}

2) POST /api/marketplace/redemption/quote
Body:
{
  provider: 'ebay'|'goldin',
  externalId: string,
  priceCents: number,
  currency: 'USD'
}
Return: Quote object with ptsMaxApplicable and a recommended default ptsApplied.

RULES (must enforce in quote):
- If user status is RESTRICTED or FROZEN => ptsMaxApplicable = 0 and reasons include "Redemption disabled"
- If debtPts > 0 => ptsMaxApplicable = 0
- ptsMaxApplicable is the MIN of:
  a) user availablePts
  b) policy cap: max % discount or max pts per transaction
  c) treasury-backed cap (available margin pool + affiliate economics)
  d) minimum cash floor (e.g., user must pay at least $X cash)
- Choose default ptsApplied as:
  min(ptsMaxApplicable, “recommendedPts”)
  where recommendedPts targets a psychologically strong savings (e.g., 10–20% off) BUT never violates caps.

Compute:
usdSavingsCents = min(priceCents - minCashFloorCents, ptsApplied * valuePerPtCents)
usdDueCents = priceCents - usdSavingsCents
Never allow usdDueCents < minCashFloorCents.

3) POST /api/marketplace/redemption/apply
(Existing apply flow)
Body:
{
  provider,
  externalId,
  priceCents,
  ptsRequested
}
Server:
- re-quotes
- clamps
- locks pts (LOCK_RESERVATION) + reserves margin (company side)
- returns final approved {ptsApplied, usdDueCents, reservationIds}

========================================================
C) BACKEND — BATCH QUOTES (IMPORTANT FOR LIST VIEW)

Marketplace lists can show 20 listings; do NOT call quote endpoint 20 times individually.

Implement:
POST /api/marketplace/redemption/quote-batch
Body:
{
  items: [{ provider, externalId, priceCents, currency }]
}
Return:
{ quotes: Record<string, Quote> }
Key by `${provider}:${externalId}`

Cache quotes for 60–120 seconds per user to reduce DB load.

E) APPLY MODAL (CONVERSION ENGINE)

When user clicks "Apply PackPTS":
Open modal with:
- Listing summary + image + cash price
- Slider (or preset buttons) to choose how many PackPTS to apply:
    Presets: 25% / 50% / Max

- Show:

    Savings in USD
    Remaining PackPTS after purchase
- Confirm button:
    “Continue to eBay with PackPTS Applied”

On confirm:
- Call /api/marketplace/redemption/apply (locks pts + reserves margin)
- Return a “tracked outbound URL” to redirect user to eBay/Goldin listing (with affiliate params if you have them)
- Show confirmation screen:
    “Reserved: {ptsApplied} PackPTS — complete purchase in the next 10 minutes”
- Start timer; on timeout/cancel -> release reservations.
