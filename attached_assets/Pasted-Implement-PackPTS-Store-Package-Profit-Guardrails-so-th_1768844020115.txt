Implement “PackPTS Store Package Profit Guardrails” so that any PackPTS package (USD → PackPTS) created/edited by an Admin is automatically validated for profitability. The system must block or warn when a package’s implied USD-per-PackPTS ratio would be unprofitable or economically unsafe, considering payment processing fees, app store fees, and the company’s minimum gross margin requirement.

STACK (do not change)
Frontend: React 18 + Vite + TypeScript + Tailwind + shadcn/ui + TanStack React Query + Wouter
Backend: Express + TypeScript (tsx dev)
DB: Postgres (Neon) + Drizzle ORM + migrations
Auth: Passport + express-session
Payments: Stripe (web). (If later mobile IAP, include Apple/Google fee profiles too.)

CORE IDEA
When selling PackPTS, you are selling a stored-value unit. Each package must:
1) Cover variable fees (Stripe / IAP)
2) Fund the company margin pool (treasury)
3) Establish a conservative “max redemption cost” per PackPTS

We will implement a guardrail that evaluates each package and:
- HARD BLOCK if it violates minimum margin (must be profitable)
- WARN if it’s above risk thresholds (too generous; liability heavy)

DEFINITIONS / MATH (MUST IMPLEMENT)
For a package:
- price_cents = what the user pays (USD)
- pts = PackPTS granted
- fee_rate = variable processor rate (e.g., Stripe 2.9% => 0.029)
- fee_fixed_cents = fixed fee per txn (e.g., 30 cents)
- platform_fee_rate = app store fee rate (web=0, Apple/Google typically 0.15–0.30) // configurable per channel
- min_margin_rate = minimum gross margin required on PackPTS sales (default 0.30)
- reserve_rate = percentage of post-fee revenue routed into margin_pool (default 1.0 for simplicity, but keep configurable)
- v_usd_per_pt = implied USD value per PackPTS for “economic ceiling”

Compute:
1) total_fees_cents =
   round(price_cents * fee_rate) + fee_fixed_cents
 + round(price_cents * platform_fee_rate)

2) net_revenue_cents =
   price_cents - total_fees_cents

3) gross_margin_rate =
   net_revenue_cents / price_cents

HARD RULE (BLOCK):
gross_margin_rate >= min_margin_rate

If not, block creation/edit OR allow save with explicit override flag (admin-only) depending on policy.

4) implied_value_per_pt_cents =
   net_revenue_cents / pts

This is the max “economic” value you can ever afford to give per point if the user eventually redeems everything.

WARN RULES (do NOT block, but show “Are you sure?”):
- implied_value_per_pt_cents > policy.max_value_per_pt_cents (too generous)
- pts is extreme vs package tier ladder (outlier package)
- gross_margin_rate is within warning band (e.g., < min_margin_rate + 0.05)

BACKEND IMPLEMENTATION
1) Guardrail engine module
Create: src/services/store/packageGuardrails.ts
Export:
- evaluatePackage({price_cents, pts_grant, channel}): returns {decision, reasons, computed}
Uses active store_package_policy + active store_fee_profiles[channel].

2) Admin API
A) POST /api/admin/store/packages
Body: { sku, name, priceCents, ptsGrant, channel }
Flow:
- compute guardrail evaluation
- if decision=BLOCK -> return 422 with reasons + computed
- if decision=WARN -> allow save but require confirm=true OR second endpoint to confirm
- save store_products
- write store_package_validations row (PASS/WARN)

B) PUT /api/admin/store/packages/:id
Same logic as create.

C) POST /api/admin/store/packages/:id/override
Only if policy.allow_override=true AND admin role.
Writes decision=OVERRIDE with note.

D) GET /api/admin/store/packages/:id/validation
Returns computed fields + last decision.

3) Frontend Admin UX
Admin “Create/Edit Package” form:
- inputs: price, pts_grant, channel
- live “Profit Preview” panel that calls:
  POST /api/admin/store/packages/preview (or reuse evaluate endpoint)
Show:
- total fees
- net revenue
- gross margin %
- implied USD per PackPTS
- decision badge: PASS / WARN / BLOCK
- reasons list

If WARN:
- show modal: “This package is economically risky. Type CONFIRM to save.”
If BLOCK:
- disable save and show computed explanation.

4) When a user purchases a package (Stripe checkout completed)
- Create margin_ledger entry of type PACKPTS_SALE for:
  margin_cents = net_revenue_cents * reserve_rate
- Credit user PackPTS via ledger (EARN/PURCHASE_CREDIT)
- This keeps the treasury-backed system consistent.
