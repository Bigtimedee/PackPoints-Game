Implement CardHedge “Get Card Details by ID” integration for PackPTS using:

POST https://api.cardhedger.com/v1/cards/card-details

This endpoint fetches detailed metadata, pricing, sales history, and image URLs for a specific trading card by its CardHedge card_id.

STACK (do not change)
Frontend: React 18 + Vite + TypeScript + Tailwind + shadcn/ui + TanStack React Query + Wouter
Backend: Express + TypeScript (tsx dev)
DB: Postgres (Neon) + Drizzle ORM
Auth: Passport + express-session

SECURITY REQUIREMENTS
- API key must NEVER be exposed to frontend
- All CardHedge calls must be server-side
- Enforce HTTPS only
- Add rate limiting on public endpoints

ENV VARS (.env.example)
CARDHEDGE_BASE_URL=https://api.cardhedger.com
CARDHEDGE_API_KEY=
CARDHEDGE_TIMEOUT_MS=10000
CARDHEDGE_CACHE_TTL_SECONDS=600

REQUEST FORMAT (MUST MATCH API)
POST /v1/cards/card-details
Headers:
- Content-Type: application/json
- X-API-Key: <CARDHEDGE_API_KEY>

Body:
{
  "card_id": "string",
  "raw_images_only": boolean (optional)
}

RESPONSE STRUCTURE (EXPECTED)
Response returns:

{
  "cards": [
    {
      "description": string,
      "player": string,
      "set": string,
      "number": string,
      "variant": string,
      "card_id": string,
      "image": string,
      "category": string,
      "category_group": string,
      "set_type": string,
      "7 Day Sales": number,
      "30 Day Sales": number,
      "rookie": boolean,
      "gain": number,
      "prices": [ ... ]
    }
  ]
}

BACKEND IMPLEMENTATION

1) Create CardHedge client module

File: src/services/cardhedge/cardDetails.ts

Implement:

async function fetchCardDetails(cardId: string, rawImagesOnly?: boolean)

Behavior:
- POST to /v1/cards/card-details
- Add headers:
  X-API-Key
  Content-Type application/json
- Add timeout protection
- Normalize errors into:
  { statusCode, message }

2) Add caching layer

Implement cache table OR in-memory cache:

Key:
"card-details:" + cardId + ":" + rawImagesOnly

TTL:
CARDHEDGE_CACHE_TTL_SECONDS

3) Create API endpoint

GET /api/cardhedge/card/:cardId

Query params:
- rawImagesOnly=true|false (optional)

Behavior:
- Validate cardId exists
- Check cache
- Call fetchCardDetails if not cached
- Normalize response to internal shape:

Return:

{
  cardId,
  description,
  player,
  set,
  number,
  variant,
  category,
  categoryGroup,
  setType,
  imageUrl,
  rookie,
  sales7d,
  sales30d,
  gain,
  prices,
  raw
}

IMPORTANT:
- Preserve full original payload inside "raw" field for future compatibility.

4) Database optional persistence (recommended)

Create table:

card_details_cache

Fields:
- card_id text primary key
- payload jsonb
- fetched_at timestamp
- expires_at timestamp

Use this table for persistent caching.

5) Admin testing endpoint

Add admin-only UI route:

/admin/cardhedge-card

Features:
- Input field: card_id
- Toggle: raw_images_only
- Button: Fetch Card
- Display:
  - Card image
  - Player
  - Set + Number + Variant
  - Rookie badge
  - 7-day sales
  - 30-day sales
  - Gain %
  - JSON inspector for raw payload

6) Gameplay integration hook

Expose helper:

getCardImageForGameplay(cardId)

Rules:
- Prefer raw_images_only=true
- Fallback to standard image if none returned
- Cache aggressively (cards don’t change often)

Used when:
- Rendering PackPTS quiz cards
- Creating gameplay packs

7) Rate limiting

Apply limits:
- Public endpoint: 30 requests/min/IP
- Admin endpoint: 60 requests/min/admin

8) Error handling UX

If CardHedge fails:
- Return cached data if available
- Otherwise return:
  503 CARD_DATA_TEMPORARILY_UNAVAILABLE

Frontend should display:
“Card image temporarily unavailable — retrying”

TESTS REQUIRED

- Valid card_id returns normalized payload
- raw_images_only flag passes through
- Cache hit avoids API call
- Invalid card_id returns 422
- Timeout handled gracefully
- API key missing fails fast at startup
