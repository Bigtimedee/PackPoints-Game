STEP 2 — Apply Known iOS Safari Fixes for Radix Select
Implement ALL of the following changes to the Card Set dropdown:

A) Ensure the dropdown content is portaled to <body>
- Use Radix Select’s default portal behavior OR explicitly wrap content in <Select.Portal>.
- This avoids parent containers with overflow:hidden or transform breaking clicks.

B) Use position="popper" and avoid transformed ancestors
- For shadcn SelectContent, set:
  <SelectContent position="popper" sideOffset={8} avoidCollisions={true} />
- Ensure no parent container of the trigger has CSS transform (transform creates a new containing block in Safari and can break fixed positioning).

C) Fix z-index + pointer events
- Ensure SelectContent has a very high z-index:
  className="z-[9999]"
- Ensure no parent overlay div is capturing pointer events (pointer-events: none vs auto).
- Verify the trigger is not inside a container with `pointer-events: none`.

D) Fix iOS touch/pointer close bug
iOS Safari sometimes closes popovers immediately due to pointerdown/outside logic.
- Prefer opening on pointerdown:
  Add onPointerDown={e => e.preventDefault(); /* keep focus stable */} on the Trigger
  Then manually open by controlling state (see STEP 3).
- Alternatively, use onClick rather than onPointerDown for iOS only.

E) Disable scroll locking on iOS (if using Dialog/Sheet)
If the dropdown is inside a Dialog/Sheet, Radix scroll-lock can interfere.
- Ensure SelectContent is NOT inside an element with `-webkit-overflow-scrolling: touch` conflicts.
- Add:
  <SelectContent
    onCloseAutoFocus={(e) => e.preventDefault()}  // prevents focus-jump issues
  />

F) Provide a fallback to native <select> on iOS if needed
If Radix Select still fails, detect iOS Safari and render a native <select> for Card Set only.
Detection:
- iOS: /iPad|iPhone|iPod/.test(navigator.userAgent)
- Safari: /Safari/.test(UA) && !/CriOS|FxiOS/.test(UA)

STEP 3 — Convert Card Set dropdown to a Controlled Select (Most Reliable)
Make the Card Set dropdown controlled so we can manage open state and avoid iOS event edge cases.

- Add state:
  const [open, setOpen] = useState(false);
  const [value, setValue] = useState<string>("");

- Render:
  <Select open={open} onOpenChange={setOpen} value={value} onValueChange={(v) => { setValue(v); setOpen(false); }}>
    <SelectTrigger
      onTouchStart={(e) => { /* iOS: ensure it opens */ setOpen(true); }}
      onClick={() => setOpen(true)}
    >
      ...
    </SelectTrigger>

    <SelectPortal>
      <SelectContent
        position="popper"
        sideOffset={8}
        className="z-[9999] max-h-[60vh] overflow-y-auto"
        onCloseAutoFocus={(e) => e.preventDefault()}
      >
        ...
      </SelectContent>
    </SelectPortal>
  </Select>

- Ensure content is scrollable with a max height on small screens:
  max-h-[60vh] overflow-y-auto
