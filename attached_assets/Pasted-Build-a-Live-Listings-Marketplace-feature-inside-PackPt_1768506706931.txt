Build a “Live Listings Marketplace” feature inside PackPts that pulls current trading card options from external marketplaces (starting with eBay via official APIs, and scaffolding Goldin as a partner feed later), and routes users through tracked outbound links. Integrate this into the existing app sections:

- Marketplace = discovery + browse live external listings (no PackPTS redemption applied here)
- Store = PackPTS redemption rules + generate tracked deep links and explain what the user gets

STACK (do not change)
Frontend: React 18 + Vite + TypeScript + Tailwind + shadcn/ui + TanStack React Query + Wouter
Backend: Express + TypeScript + Passport + express-session
DB: Postgres (Replit Neon) + Drizzle ORM
Email: Nodemailer (existing)
Payments: Stripe (existing)

GOALS
1) Add a Marketplace section that shows “current listings” for a queried card/player/set with near-real-time results.
2) Implement an eBay integration using official APIs:
   - server-side search endpoint
   - short TTL caching
   - normalized listing schema
   - outbound redirect endpoint that applies affiliate tracking (EPN-style) and logs clicks
3) Add a Goldin integration “stub” with a DB-backed curated feed AND a clean interface for a future partner/private feed.
4) Wire the UX so users can:
   - discover listings in Marketplace
   - see PackPTS-related messaging in Store
   - click “Buy on eBay” / “View on Goldin” via tracked redirects
5) Include freshness indicators (“last updated”) and safe rate limiting.

NON-NEGOTIABLE CONSTRAINTS
- API keys/credentials must remain server-side only.
- Do not scrape websites. Use eBay official APIs. For Goldin, implement a partner-feed interface + curated items (manual entry) until a supported feed exists.
- Cache external results with TTL to reduce API load (default 5 minutes; configurable).
- All outbound links must go through our backend redirect endpoint to allow tracking and future attribution/rebate logic.
- Must be safe with auth: browsing is public/optional, but click tracking can be anonymous or authenticated; preference is track both.
- Add affiliate disclosure text in UI where outbound affiliate links exist.

BACKEND IMPLEMENTATION (Express)
A) Create a marketplace service layer
- src/services/marketplace/types.ts
  Define a normalized Listing type:
  Listing = {
    source: 'ebay'|'goldin',
    listingId: string,
    title: string,
    imageUrl?: string,
    price: { amount: number, currency: string } | null,
    shipping?: { amount?: number, currency?: string } | null,
    condition?: string | null,
    endTime?: string | null,
    url: string,
    lastUpdated: string
  }

- src/services/marketplace/ebay.ts
  Implement:
  - getEbayAccessToken(): server-side OAuth client-credentials (cached in memory with expiry)
  - searchEbayListings(params): calls official eBay Browse search endpoint
  - normalize response into Listing[]

- src/services/marketplace/goldin.ts
  Implement:
  - getCuratedGoldinListings(query/tags): reads from goldin_curated_listings
  - define interface for future partner feed: fetchGoldinPartnerFeed()

B) Caching layer
- Implement getOrSetCache(source, cacheKey, ttlSeconds, fetcher):
  - read marketplace_cache
  - if not expired return payload
  - else fetch fresh, store payload with expires_at, return
Default TTL: 300 seconds. Allow override.

C) Public search API endpoints
1) GET /api/marketplace/search
Query params:
- q (string, required)  e.g., "1987 Topps Kirby Puckett"
- source (optional: 'ebay'|'goldin'|'all', default 'all')
- limit (optional, default 20)
- sort (optional: 'relevance'|'priceAsc'|'priceDesc'|'endingSoon')
Behavior:
- Build a cacheKey from source+q+limit+sort
- For ebay: fetch via ebay.ts (cached)
- For goldin: return curated results (and later partner feed)
- Return: { listings: Listing[], lastUpdated, sourceBreakdown }

2) GET /api/marketplace/trending (optional)
- Return most searched queries from event log (if exists) or from external_listings_snapshot.

D) Outbound redirect endpoints (tracked links)
1) GET /out/ebay/:listingId
- Accept query param `u` for the raw destination URL OR look up URL in a short-lived signed token.
- Preferred: return listings with a signed `outToken` that encodes source, listingId, destinationUrl, expiresAt.
- Validate outToken server-side, log outbound_clicks, then redirect (302) to destination.
- Apply affiliate tracking if configured:
  - Append EPN parameters and optional custom id derived from user id (HMAC with EPN_CUSTOM_ID_SALT), never exposing raw user id.

2) GET /out/goldin/:listingId
- Similar behavior: log click, redirect to destination_url from curated listing.

E) Rate limiting + safety
- Rate limit /api/marketplace/search by IP (and user if authed): e.g., 30/min.
- Validate q length, strip dangerous input, enforce max limit.
- Never expose eBay tokens to client.

F) Admin endpoints for Goldin curated feed
- POST /api/admin/goldin/listings (create/update)
- GET /api/admin/goldin/listings
- Protected by admin role.
