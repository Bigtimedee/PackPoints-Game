PART C — DAILY 5 CHALLENGE SYSTEM (IN-APP)
========================================================

C1) Create a new game mode: DAILY5
Rules:
- Once per day per user.
- All users get the same 5 cards for that day (per sport/category).
- Start time: default 8:00 PM Eastern, configurable.
- The “Daily 5” uses a deterministic seed so the card selection is reproducible and can be audited.
- Score: total correct + time bonus (optional).
- Leaderboard: daily reset. Show top 100.

C2) Database tables
1) daily_challenges
- id (uuid)
- date (YYYY-MM-DD unique)
- mode (text default 'DAILY5')
- set_id (fk to playable set) OR query definition snapshot
- seed (text)
- starts_at (timestamp)
- ends_at (timestamp)
- status (enum: SCHEDULED, ACTIVE, CLOSED)
- created_at

2) daily_challenge_cards
- id (uuid)
- daily_challenge_id (fk)
- position (int 1-5)
- card_id (fk to cards table)
- created_at

3) daily_challenge_entries
- id (uuid)
- daily_challenge_id (fk)
- user_id (fk)
- started_at
- completed_at
- score (int)
- correct_count (int)
- time_ms (int)
- answers (json) // store user answers, card positions
- created_at
Unique constraint: (daily_challenge_id, user_id)

C3) APIs
- GET /api/daily5/status
  returns today’s challenge status, start/end times, and whether user has played.

- POST /api/daily5/start
  creates entry, returns card payloads (masked details as required by PackPTS game design)

- POST /api/daily5/answer
  records answer for a card position; server validates; returns correctness.

- POST /api/daily5/finish
  finalizes score; returns summary + rank (approx).

- GET /api/daily5/leaderboard?date=YYYY-MM-DD
  returns top entries for date; default today.

C4) Selection logic
- When the challenge is created (scheduled job):
  - Pull candidate cards from the chosen playable set where:
    - image is not silhouette/placeholder
    - has required fields
  - Randomize deterministically using seed: hash(date + set_id + SECRET_SALT)
  - Take first 5 cards
  - Save to daily_challenge_cards
This prevents “autonomous shrinkage” from affecting historical Daily 5 cards because they’re snapshotted.

C5) UI
- Add a “Daily 5” entry point in Play:
  - If before start time: show countdown + notify option
  - If active and not played: Start button
  - If played: show score + rank + leaderboard

- Add Daily 5 leaderboard page with shareable “I ranked #X” card.

C6) Growth agent integration
- The Growth Agent reads:
  - Today’s challenge theme + start time for announcement posts
  - Yesterday’s winners + top scores for recap posts
- Auto-generates:
  - Discord announcement at start
  - Discord recap after close
  - Manual Publishing Queue items for other platforms
